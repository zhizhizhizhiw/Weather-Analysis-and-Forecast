# 大连市天气数据分析与预测 —— 技术栈全解析

---

## 总览

本项目按数据流向分为三个阶段：网络爬虫（数据采集）→ 数据清洗（结构化处理）→ 可视化分析 + 时间序列预测。每个阶段涉及不同的技术栈，下面逐一展开。

## 一、Python（编程语言）

**是什么：** Python 是一门解释型、动态类型的通用编程语言，以简洁的语法和丰富的第三方库生态著称，是数据科学领域的事实标准语言。

**怎么用：** 作为整个项目的"胶水语言"，串联所有模块。从爬虫到数据处理到绑图到建模，全部用 Python 实现。

**在项目中的作用：** 项目用 Jupyter Notebook（.ipynb）作为开发环境，分三个 Notebook 对应三个阶段的代码。Notebook 的优势是可以逐 cell 运行、中间变量持久化、图文混排，适合数据分析这种需要反复试探和调参的工作流程。

## 二、requests（HTTP 请求库）

**是什么：** requests 是 Python 最流行的 HTTP 客户端库，封装了底层的 socket 和 urllib，用接近自然语言的方式发起网络请求。

**核心用法：** `requests.get(url, headers=..., timeout=...)` 发送 GET 请求，返回 response 对象，其中 `response.text` 是响应体的文本内容，`response.encoding` 可以手动指定编码。

**在项目中的作用：** 向天气后报网站按月发起 48 次 HTTP 请求（2022-2025 年，每年 12 个月），获取每个月的天气历史页面 HTML。

**遇到的难点：**
- **反爬机制：** 网站检测到裸 Python 请求会拒绝连接。我的解决方案是构造一个完整的浏览器 User-Agent 头，伪装成 Chrome 浏览器访问。
- **页面不存在处理：** 2025 年 8 月之后数据未生成，请求返回空页面。代码中用 `table is None` 判空跳过，保证爬虫整体不崩溃。最终 48 个月中 43 个月有数据，5 个月返回空（2025/08-12）。
- **请求频率控制：** 每次请求后 `time.sleep(0.5)`，既防止被目标服务器封 IP，也遵循爬虫的基本道德规范。
- **编码问题：** 网站中文编码有时不稳定，手动设置了 `response.encoding = "utf-8"`。

## 三、BeautifulSoup4（HTML 解析库）

**是什么：** BeautifulSoup 是 Python 的 HTML/XML 解析库，能从不规范的网页源码中提取结构化的数据。与正则表达式解析 HTML 相比，BeautifulSoup 更可靠——它理解 DOM 树结构，容错性强。

**核心用法：**
- `BeautifulSoup(html, "html.parser")` 把 HTML 字符串解析成文档树
- `soup.find("table", class_="weather-table")` 按标签名+CSS 类名定位目标元素
- `table.find("tbody").find_all("tr")` 逐层遍历 DOM 树找到所有行
- `cols[i].get_text(strip=True)` 提取单元格文本，`strip=True` 去除首尾空白

**在项目中的作用：** 解析天气后报网站的月度天气表格。每个页面有一个 `<table class="weather-table">`，每行 4 列（日期、天气状况、温度、风力）。BeautifulSoup 负责定位这个表格、遍历所有行、提取每列文本。

**遇到的难点：**
- **表格行长度不一致：** 有些行可能缺少列，我加了 `if len(cols) < 4: continue` 的防御性判断。
- **文本中藏有多余空白：** 天气字段 `"多云 /  晴"` 中有不规则空格，影响后续清洗。这个问题在 pandas 阶段才完全解决。

**为什么不用 Scrapy：** Scrapy 是异步爬虫框架，适合大规模数据采集。本项目只有 48 个页面、1332 条记录，requests + BeautifulSoup 足够了，过度设计反而增加复杂度。

## 四、pandas（数据分析库）

**是什么：** pandas 是 Python 最核心的数据处理库，提供 DataFrame（二维表）和 Series（一维序列）两种数据结构，支持分组、聚合、透视、时间序列操作等。

**在项目中的作用——贯穿全部三个阶段：**

### 4.1 数据落盘

`df.to_csv("dalian_weather.csv", index=False, encoding="utf-8")` 把爬取结果保存为 CSV。csv 是纯文本格式，Excel 和任何编程语言都能直接打开，比存成 pickle 或数据库更适合这种规模的数据。

### 4.2 数据清洗（核心环节）

这是整个项目里逻辑密度最高的代码块。原始数据的三个核心字段都是"白天/夜间"复合格式：

| 字段 | 原始值 | 需拆分出 |
|------|--------|----------|
| 天气 | `多云 /  晴` | weather_day, weather_night |
| 温度 | `7℃/-7℃` | temp_high, temp_low |
| 风力 | `北风 5-6级 / 北风 5-6级` | wind_day_level, wind_night_level |

**具体操作：**

用 `str.split('/', expand=True)` 按 `/` 分隔符将天气和风力字段拆成白天/夜间两列。但这是粗拆分，还需要精提取。

用 `str.extract(r'(\d+-\d+)')` 从 `"北风 5-6级"` 中提取出风力等级 `"5-6"`。extract 是 pandas 内建的正则接口，返回匹配到的第一个捕获组。

用 `str.extract(r'(-?\d+)℃/(-?\d+)℃')` 同时提取最高和最低温度。其中 `-?` 处理负温度（大连冬天经常零下），`℃` 前的 `(-?\d+)` 捕获数值部分。

最后用 `astype(float)` 把温度转为数值类型，`dropna(subset=["temp"])` 过滤温度缺失的行。

### 4.3 数据聚合统计

- `groupby([date.dt.year, date.dt.month])['temp_avg'].mean()`：按年+月分组，计算月均气温
- `groupby(['month', 'wind_day_level']).size().unstack()`：按月份+风力等级交叉统计天数，unstack 把风力等级从行展开为列，形成 12×N 的矩阵
- `.div(3)`：除以 3 年得到月均值

### 4.4 日期处理

`pd.to_datetime("2022年01月01日", format="%Y年%m月%d日")` 把中文格式日期转为 Python datetime 对象。之后就能用 `.dt.year`、`.dt.month` 提取年月做分组统计了。参数 `errors="coerce"` 表示格式不匹配时填 NaT（Not a Time）而不是报错。

### 遇到的难点：

- **清洗后仍有脏数据：** 天气字段中有全角空格 `　` 和多余半角空格，导致 `"晴"` 和 `"晴 "` 被当成两种天气类型。在 DataAnalyse 中额外加了 `str.replace('　', '').str.strip()` 的处理。
- **类型转换陷阱：** 正则提取出来的值是字符串，不转 float 就没法做数值运算（如求平均温度）。同理，日期不做 to_datetime 就没法按月分组。
- **unstack 的理解门槛：** groupby 后得到的是多层索引，unstack 把第二层索引变为列名。这个操作如果不熟悉，很难理解为什么数据突然从一列变成了一个矩阵。

## 五、正则表达式（re，文本模式匹配）

**是什么：** 正则表达式是一种用特殊语法描述字符串模式的工具，用于查找、提取、替换文本中符合特定规则的部分。Python 内建 re 模块支持，pandas 的 `.str.extract()` 内部也是调用 re。

**在项目中的作用：**

- `r'(-?\d+)℃/(-?\d+)℃'`：同时提取最高温度和最低温度。括号定义了两个捕获组，`-?` 匹配可选负号
- `r'(\d+-\d+)'`：提取风力等级如 `"5-6"`、`"3-4"`

**为什么不用 str.split：** 温度 `"7℃/-7℃"` 如果只用 split('/')，得到的是 `["7℃", "-7℃"]`，还需要再去掉 `"℃"`。而用正则一步到位，直接拿到数值 `"7"` 和 `"-7"`。正则虽然写法不直观，但处理半结构化字符串比反复 split + replace 更可靠。

**遇到的难点：** 正则语法本身容易写错。比如 `(-?\d+)` 如果不加 `?`，`-7℃` 就匹配不到——因为 `-` 不算 `\d` 的一部分。另外 pandas extract 必须有捕获组（括号），否则不返回结果，这是一个常见的踩坑点。

## 六、matplotlib（数据可视化库）

**是什么：** matplotlib 是 Python 事实标准级的静态绑图库，风格对标 MATLAB。功能涵盖折线图、柱状图、散点图、饼图、热力图等几乎所有常见图表类型。

**核心概念：** Figure（画布）→ Axes（子图，即一个坐标系）→ 绑图元素（plot/scatter/bar 等）。每个元素都可以精细控制：颜色、线型、透明度、标签、图例位置等。

**在项目中的作用：**

1. **月平均气温折线图：** `monthly.plot(marker='o')` 以月份为 x 轴、温度为 y 轴，4 条折线对应 4 个年份。用 `ax.text()` 在每个数据点上标注数值。
2. **每日气温变化图（含异常检测）：** `plt.plot()` 绑日均气温橙色细线和 7 日滑动平均绿色虚线；`plt.scatter()` 标注高温异常（红色）和低温异常（蓝色）；`plt.axvline()` 画年份分隔竖线。
3. **风力等级堆叠柱状图：** `wind_monthly.plot(kind='bar', stacked=True)`，x 轴 12 个月，每种风力等级一种颜色堆叠。
4. **天气状况堆叠柱状图（白天+夜间各一张）：** 同上，按天气类型堆叠。

**配置要点：**
- `plt.figure(figsize=(12, 6))` 控制画布大小，避免图表被压缩变形
- `plt.tight_layout()` 自动调整元素间距，防止标签被截断
- `plt.grid(axis='y', linestyle='--', alpha=0.5)` 添加半透明虚线网格，便于读数

**遇到的难点：**
- **中文乱码：** matplotlib 默认字体不含中文字符，图表标题和标签全部显示为方框。需要两行配置解决：`plt.rcParams['font.sans-serif'] = ['SimHei']` 指定中文字体 + `plt.rcParams['axes.unicode_minus'] = False` 防止负号也变方框。这是几乎每个中文 Python 数据从业者都会踩的坑。
- **数据标注重叠：** 4 条曲线在部分月份数值接近，标注文字互相覆盖。靠调整字体大小（`fontsize=8`）和对齐方向（`va='bottom'`）部分缓解，但没有完美解决——这在真实项目中也很常见。

## 八、时间序列预测相关库

### 8.1 statsmodels（统计建模库）

**是什么：** statsmodels 是一个专注于统计推断和计量经济学的 Python 库，提供了丰富的统计模型（线性回归、ARIMA、VAR、状态空间模型等）。它的定位和 scikit-learn 互补——statsmodels 偏"解释性建模"（看 p 值、置信区间），scikit-learn 偏"预测性建模"。

**核心用法：**
```python
from statsmodels.tsa.statespace.sarimax import SARIMAX
model = SARIMAX(data, order=(1, 0, 3))
result = model.fit()
result.fittedvalues  # 获取拟合值
```

**在项目中的作用：** 构建 SARIMAX 时间序列模型（Seasonal AutoRegressive Integrated Moving Average with eXogenous variables）。但本项目没有用到外生变量（eXogenous），实际上跑的是一个简化版——普通 ARIMA。用 auto_arima 确定的阶数 (1,0,3) 作为 order 参数传入。

### 8.2 ARIMA 模型阶数 (1,0,3) 的含义是什么？

ARIMA(p, d, q) 三个参数分别代表：

- **p=1（自回归阶数）：** 用前 1 天的温度来预测今天的温度。p=1 意味着模型认为今天的温度主要受昨天的温度影响。
- **d=0（差分阶数）：** d=0 表示原序列已经平稳，不需要差分。温度数据在固定范围内波动（-10℃ 到 30℃），不像股票价格那样有增长趋势，所以 d=0 是合理的。
- **q=3（移动平均阶数）：** 考虑前 3 天的预测误差来修正当前预测。q=3 意味着模型会参考最近 3 天的"预测偏差"来调整今天的输出。

**模型在做什么（直观理解）：** 预测明天的温度，可以看昨天的温度（p=1），再结合最近几天的预测失误（q=3）做微调。因为温度没有长期上升或下降趋势，不需要差分（d=0）。

### 8.3 pmdarima（ARIMA 自动选参库）

**是什么：** pmdarima 是 statsmodels 的上层封装，核心功能是 `auto_arima`——自动搜索最优的 ARIMA 参数组合，省去人工尝试各种 (p,d,q) 组合的麻烦。

**核心用法：**
```python
from pmdarima import auto_arima
auto_model = auto_arima(data, seasonal=True, m=12, trace=True)
print(auto_model.order)  # 输出 (1, 0, 3)
```

**参数含义：**
- `seasonal=True, m=12`：开启季节性模式，12 表示年周期（12 个月一轮回）
- `trace=True`：打印每一步的 AIC 值，方便观察模型收敛过程
- `stepwise=True`：用逐步搜索而非全面网格搜索，节省计算时间
- `max_p=3, max_q=3`：限制搜索范围

**选参原理：** auto_arima 在给定范围内尝试所有可能的 (p,d,q) 组合，对每组参数拟合 ARIMA 模型，计算 AIC（赤池信息准则）。AIC 同时衡量模型拟合度（残差大小）和复杂度（参数个数），AIC 最小的参数组合胜出。这就像找一个既解释数据充分又不至于过拟合的"刚刚好"的模型。

### 8.4 scikit-learn（机器学习库，评估指标用）

**是什么：** scikit-learn 是 Python 最主流的机器学习库，覆盖分类、回归、聚类、降维、模型选择和评估。

**在项目中的作用：** 只用到了它的评估指标——`mean_squared_error()`（均方误差）和 `mean_absolute_percentage_error()`（平均绝对百分比误差）。没有用到任何机器学习模型。

### 遇到的难点：

**MAPE 爆炸问题（重要 Bug）**

模型跑完，MSE 是 4.87（合理），但 MAPE 显示 16662823273633796%，一个天文数字。最开始以为是代码写错了，排查了才发现是数学原因。

MAPE 的计算公式是：`(|实际值 - 预测值| / 实际值) 的平均值 × 100%`。关键问题——大连冬天温度在 0℃ 附近波动，当某天的实际温度接近 0℃ 时，分母趋近于零，导致单日误差暴涨，进而把整体均值拉到天文数字。

**根本原因：** MAPE 天生不适合含零及接近零的数据。这不是代码错误，而是"选择的评估指标和数据特性不匹配"。

**解决方案：** 放弃 MAPE，改用 MSE 作为唯一评估指标。面试时还可以延伸说：如果要保留百分比类的指标，可以用 sMAPE（对称版本，分母是实际值+预测值的绝对值和的一半，永远不为零），但 sMAPE 本身也有争议。

**教训：** 跑出结果不等于结果正确。一个靠谱的数据分析者必须能判断数字是否合理，并在不合理时有能力溯源到根因。这个 bug 本身就是一个很好的面试故事。

## 项目各技术栈的层次关系

```
┌─────────────────────────────────────────────────┐
│                   Jupyter Notebook              │
│                  （开发环境/胶水层）               │
├───────────────┬───────────────┬─────────────────┤
│   爬虫层       │   分析层       │   预测层          │
│  requests     │  pandas       │  pandas          │
│  BeautifulSoup│  matplotlib   │  numpy           │
│  正则表达式    │  numpy        │  statsmodels     │
│  time         │               │  pmdarima        │
│               │               │  scikit-learn    │
│      ↓        │      ↓        │       ↓          │
│  CSV 文件 ───→ cleaned_weather.csv ──→ 预测结果    │
└─────────────────────────────────────────────────┘
```

数据从 requests 流向 BeautifulSoup，经正则清洗后变为 CSV，pandas 读取后做分组聚合，matplotlib 绑制图表，statsmodels/pmdarima 做时间序列拟合。整个链路每步衔接的"接头"就是 CSV 文件——dalian_weather.csv 是爬虫产出，cleaned_weather.csv 是清洗产出、也是分析和预测的输入。
